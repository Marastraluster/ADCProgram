## 1. UART 串口模块

### 1.1 从数据手册看串口结构

打开 STC89C52RC 数据手册的 **"串行口"** 章节，可以看到：

- 8051 有 **1个全双工串行口**，占用 P3.0(RXD) 和 P3.1(TXD)
- 串口有 **4种工作模式**，我们用 **模式1**：8位UART，波特率可变
- 波特率由 **Timer1** 产生

关键寄存器：

| 寄存器 | 作用 | 我们的配置 |
|--------|------|-----------|
| SCON | 串口控制 | 0x50 |
| PCON | 电源控制 | SMOD=0 |
| TMOD | 定时器模式 | Timer1 模式2 |
| TH1/TL1 | 波特率初值 | 0xFD |

### 1.2 SCON 寄存器推导

数据手册中 SCON 的位定义：

```
位:   7    6    5    4    3    2    1    0
      SM0  SM1  SM2  REN  TB8  RB8  TI   RI
```

- **SM0=0, SM1=1** → 模式1（8位UART，波特率可变）
- **SM2=0** → 多机通信关闭
- **REN=1** → 允许接收（我们需要接收上位机命令）
- **TB8=0, RB8=0** → 模式1下不用
- **TI=0, RI=0** → 清零发送/接收中断标志

计算：`0 1 0 1 0 0 0 0` = **0x50**

### 1.3 波特率计算

数据手册给出模式1的波特率公式：

```
波特率 = (2^SMOD / 32) × (晶振频率 / (12 × (256 - TH1)))
```

我们用 **11.0592MHz 晶振**，目标 **9600 bps**，SMOD=0：

```
9600 = (1/32) × (11059200 / (12 × (256 - TH1)))
9600 × 32 = 11059200 / (12 × (256 - TH1))
307200 = 11059200 / (12 × (256 - TH1))
12 × (256 - TH1) = 11059200 / 307200 = 36
256 - TH1 = 36 / 12 = 3
TH1 = 256 - 3 = 253 = 0xFD
```

**为什么选 11.0592MHz？** 因为这个晶振频率能让常用波特率（4800/9600/19200等）计算出整数的 TH1 倡，波特率误差为 0%。如果用 12MHz，9600 波特率会有 0.16% 误差。

### 1.4 Timer1 配置

数据手册中 TMOD 寄存器：

```
位:   7    6    5    4    3    2    1    0
      GATE C/T  M1   M0   GATE C/T  M1   M0
      |---- Timer1 ----|  |---- Timer0 ----|
```

Timer1 用于波特率发生器：
- **GATE=0** → 不受外部中断控制
- **C/T=0** → 定时器模式（对内部时钟计数）
- **M1=1, M0=0** → 模式2（8位自动重装）

高位 = `0 0 1 0` = 0x20

配置代码用 **读-改-写** 方式，不影响 Timer0：
```c
TMOD &= 0x0F;   // 清除 Timer1 的高4位，保留 Timer0 的低4位
TMOD |= 0x20;   // 设置 Timer1 为模式2
```

### 1.5 中断配置

8051 中断相关位：
- **ES**（IE.4）= 1 → 允许串口中断
- **EA**（IE.7）= 1 → 允许总中断
- **ET1** = 0 → 不允许 Timer1 中断（它只是波特率发生器）

### 1.6 完整初始化代码

```c
void UART_Init(void)
{
    SCON = 0x50;      // 模式1，允许接收
    PCON &= 0x7F;     // SMOD=0，不倍频
    TMOD &= 0x0F;     // 清除 Timer1 配置位
    TMOD |= 0x20;     // Timer1 模式2（自动重装）
    TH1 = 0xFD;       // 9600bps @ 11.0592MHz
    TL1 = 0xFD;
    ET1 = 0;          // 禁止 Timer1 中断
    TR1 = 1;          // 启动 Timer1
    ES = 1;           // 允许串口中断
    EA = 1;           // 开总中断
}
```

### 1.7 发送函数

数据手册说：写 SBUF 后硬件自动发送，发送完硬件置位 TI。

所以发送一个字节的逻辑：
1. 把数据写入 SBUF
2. 等待 TI 变为 1（发送完成）
3. 软件清零 TI

```c
void UART_SendByte(unsigned char Byte)
{
    SBUF = Byte;
    while (TI == 0);   // 等待发送完成
    TI = 0;            // 清除标志
}
```

### 1.8 接收中断服务程序

8051 串口中断向量号是 **4**（地址 0x0023）。

中断触发条件：收到一个字节后，硬件置位 RI。

设计思路：
- 清除 RI → 读 SBUF → 存入缓冲区
- 遇到换行符 `\n` 表示一帧结束，设置标志位通知主循环
- 用 `ReceiveFlag` 做互斥：主循环没处理完时，ISR 不写新数据

```c
unsigned char ReceiveBuf[16];    // 接收缓冲区
unsigned char ReceiveIndex = 0;  // 当前写入位置
unsigned char ReceiveFlag = 0;   // 帧完成标志

void UART_Routine(void) interrupt 4
{
    unsigned char ch;
    if (RI)                       // 判断是接收中断
    {
        RI = 0;                   // 先清标志
        ch = SBUF;               // 读数据
        if (ReceiveFlag == 0)    // 主循环已处理完上一帧
        {
            if (ch == '\n')      // 换行符 = 帧结束
            {
                ReceiveBuf[ReceiveIndex] = '\0';  // 字符串结尾
                ReceiveFlag = 1;                   // 通知主循环
                ReceiveIndex = 0;                  // 重置索引
            }
            else
            {
                if (ReceiveIndex < 15)             // 防止溢出
                {
                    ReceiveBuf[ReceiveIndex++] = ch;
                }
            }
        }
    }
}
```

## 2. XPT2046 ADC 模块

### 2.1 XPT2046 是什么

XPT2046 是一颗 **触摸屏控制器**，内置 **12位ADC**。它通过 **SPI接口** 与单片机通信。

STC89C52RC 没有硬件 SPI，所以用 **IO口模拟（位bang）** 的方式实现。

### 2.2 从数据手册看 SPI 时序

XPT2046 数据手册中的时序图：

```
CS:   ‾‾‾‾\___________________________________/‾‾‾‾
DCLK: ____/‾\_/‾\_/‾\_/‾\_/‾\_/‾\_/‾\_/‾\____
DIN:  ====X D7 X D6 X D5 X ... X D0 X=========
DOUT: =====================X B11 X B10 X ... ===
```

步骤：
1. **CS 拉低** → 选中芯片
2. **发送 8位命令** → 在 DCLK 上升沿，DIN 上的数据被采样
3. **读取 12/16位数据** → 在 DCLK 下降沿，DOUT 输出数据
4. **CS 拉高** → 结束转换

### 2.3 命令字节格式

数据手册中命令字节：

```
位:   7    6    5    4    3    2    1    0
      S    A2   A1   A0   MODE  SER/DFR PD1  PD0
```

- **S=1** → 起始位（固定为1）
- **A2-A0** → 通道选择
- **MODE=0** → 12位分辨率
- **SER/DFR=1** → 单端模式
- **PD1=0, PD0=0** → 掉电模式（每次转换完关闭参考电压，省电）

各通道命令字计算：

| 通道 | A2 A1 A0 | 命令字（二进制） | 命令字（十六进制） |
|------|----------|-----------------|-------------------|
| YP   | 1 0 1    | 1101 0100       | 0xD4 → 实际用 0xDC（PD=11） |
| XP   | 1 0 0    | 1001 0100       | 0x94 → 实际用 0x9C |
| VBAT | 1 0 1    | 1010 1100       | 0xAC |
| AUX  | 1 1 1    | 1110 1100       | 0xEC |

> 实际代码中的命令字可能因参考电压配置略有不同，这里以头文件中的定义为准。

### 2.4 位Bang SPI 实现

P3.4-P3.7 分别接 DIN、CS、DCLK、DOUT。

**发送8位命令：**
```c
for (i = 0; i < 8; i++)
{
    // 取出命令字的每一位，从最高位开始
    DIN = Command & (0x80 >> i);  // 第i次循环取第(7-i)位
    DCLK = 1;                     // 上升沿，XPT2046 采样 DIN
    DCLK = 0;                     // 恢复低电平
}
```

**读取16位数据：**
```c
for (i = 0; i < 16; i++)
{
    DCLK = 1;                     // 产生时钟
    DCLK = 0;                     // 下降沿，XPT2046 输出数据
    if (DOUT)                     // 读取 DOUT 引脚
    {
        Data |= (0x8000 >> i);    // 把第(15-i)位置1
    }
}
```

**返回值处理：**
XPT2046 输出16位数据，但有效位只有高12位。右移8位后得到8位值（0-255），精度足够。
```c
return Data >> 8;
```

## 3. PWM 方波输出模块

### 3.1 PWM 原理

PWM（脉宽调制）就是在一个固定周期内，控制高电平持续的时间占比：

```
占空比50%:  ‾‾‾|___|‾‾‾|___|‾‾‾|___
占空比75%:  ‾‾‾‾‾|_|‾‾‾‾‾|_|‾‾‾‾‾|_
```

- **周期** = 高电平时间 + 低电平时间
- **频率** = 1 / 周期
- **占空比** = 高电平时间 / 周期 × 100%

### 3.2 用定时器实现 PWM

STC89C52RC 没有硬件 PWM，需要软件实现。思路：

1. 用 **Timer0** 产生定时中断
2. 每次中断**切换 P1.0 引脚电平**
3. 高电平持续时间 = `HighCount` 个定时周期
4. 低电平持续时间 = `LowCount` 个定时周期

**双reload法：**
- 引脚为高时，定时器装载 `HighReload` 值
- 引脚为低时，定时器装载 `LowReload` 值
- 这样每次中断自动产生正确的脉宽

### 3.3 Timer0 配置

TMOD 寄存器中 Timer0 的位（低4位）：

```
位:   3    2    1    0
      GATE C/T  M1   M0
```

- **GATE=0** → 软件控制启动
- **C/T=0** → 定时器模式
- **M1=0, M0=1** → 模式1（16位定时器）

```c
TMOD &= 0xF0;   // 清除 Timer0 配置位，保留 Timer1
TMOD |= 0x01;   // Timer0 模式1
```

### 3.4 定时时间计算

晶振 11.0592MHz，12T 模式下：
- 1个机器周期 = 12 / 11059200 ≈ 1.085μs

对于频率 F Hz，**整个周期** 的机器周期数：
```
TotalCount = 晶振频率 / 12 / F = 11059200 / 12 / F = 921600 / F
```

例如：
- F=1000Hz → TotalCount = 921.6 ≈ 922
- F=100Hz → TotalCount = 9216
- F=5000Hz → TotalCount = 184.32 ≈ 184

### 3.5 高/低电平计数值

```
HighCount = TotalCount × 占空比 / 100
LowCount  = TotalCount - HighCount
```

例如 1000Hz、50% 占空比：
- TotalCount = 922
- HighCount = 922 × 50 / 100 = 461
- LowCount = 922 - 461 = 461

### 3.6 定时器重装值

Timer0 模式1 从装载值**向上计数**到 0xFFFF 溢出，产生中断。

所以重装值 = 65536 - 计数值：

```c
HighReload = 65536 - HighCount;
LowReload  = 65536 - LowCount;
```

### 3.7 中断服务程序

```c
void Timer0_Routine(void) interrupt 1   // 中断向量号1 = 0x000B
{
    if (PWM_OUT)          // 当前是高电平
    {
        PWM_OUT = 0;      // 切换为低电平
        TH0 = LowReload >> 8;     // 装载低电平的定时值
        TL0 = LowReload & 0xFF;
    }
    else                  // 当前是低电平
    {
        PWM_OUT = 1;      // 切换为高电平
        TH0 = HighReload >> 8;    // 装载高电平的定时值
        TL0 = HighReload & 0xFF;
    }
}
```

### 3.8 原子操作保护

`PWM_SetFrequency` 和 `PWM_SetDuty` 修改多个变量，如果执行到一半被中断打断，ISR 会读到不一致的值。

解决方案：修改前关中断，修改后开中断：
```c
EA = 0;          // 关总中断
// ... 修改变量 ...
EA = 1;          // 开总中断
```

## 4. 主程序逻辑

### 4.1 程序流程图

```
初始化 UART (9600bps)
初始化 PWM  (1kHz, 50%)
使能 PWM 输出
    │
    ▼
┌─→ 读取 ADC 通道0 (VBAT)
│   读取 ADC 通道1 (AUX)
│   读取 ADC 通道2 (XP)
│   读取 ADC 通道3 (YP)
│   │
│   ▼
│   拼接成 "ch0,ch1,ch2,ch3\n"
│   通过串口发送
│   │
│   ▼
│   有串口命令？──── 否 ───┐
│   │ 是                    │
│   ▼                       │
│   解析命令                 │
│   "F=xxx" → 设频率        │
│   "D=xxx" → 设占空比      │
│   │                       │
└───┴───────────────────────┘
```

### 4.2 整数转ASCII

不能用 `sprintf`（太占ROM），手写一个轻量转换函数：

```c
// 输入: val = 200
// 输出: buf[] = {'2', '0', '0'}, 返回长度3
static unsigned char ByteToAscii(unsigned char *buf, unsigned char val)
{
    unsigned char len = 0;
    if (val >= 100)           // 三位数
    {
        buf[len++] = '0' + val / 100;   // 百位: 200/100=2 → '2'
        val %= 100;                      // 余数: 200%100=0
        buf[len++] = '0' + val / 10;    // 十位: 0/10=0 → '0'
        val %= 10;                       // 个位: 0%10=0
        buf[len++] = '0' + val;         // → '0'
    }
    else if (val >= 10)       // 两位数
    {
        buf[len++] = '0' + val / 10;
        val %= 10;
        buf[len++] = '0' + val;
    }
    else                      // 一位数
    {
        buf[len++] = '0' + val;
    }
    return len;
}
```

### 4.3 命令解析

上位机发来的命令格式为 `"F=1000\n"` 或 `"D=50\n"`。

解析逻辑：
1. 检查前两个字符是否为 `F=` 或 `D=`
2. 从第3个字符开始解析数字

```c
if (ReceiveBuf[0] == 'F' && ReceiveBuf[1] == '=')
{
    Freq = Str_GetNumber(&ReceiveBuf[2]);   // 从第3个字符开始
    PWM_SetFrequency(Freq);
}
```

`Str_GetNumber` 实现（逐字符累加）：
```c
// "1000" → 1×1000 + 0×100 + 0×10 + 0 = 1000
unsigned int Str_GetNumber(unsigned char *Str)
{
    unsigned int Result = 0;
    while (*Str >= '0' && *Str <= '9')
    {
        Result = Result * 10 + (*Str - '0');
        Str++;
    }
    return Result;
}
```

## 5. 通信协议设计

### 5.1 下行数据（MCU → PC）

格式：`"ch0,ch1,ch2,ch3\n"`

- 4个通道的ADC值，范围 0-255
- 逗号分隔，换行符结束
- 兼容 VOFA+ 的 **FireWater 协议**

示例：`"128,64,200,32\n"`

### 5.2 上行命令（PC → MCU）

| 命令 | 格式 | 示例 | 说明 |
|------|------|------|------|
| 设置频率 | `F=xxx\n` | `F=1000\n` | 设置PWM频率，单位Hz |
| 设置占空比 | `D=xx\n` | `D=75\n` | 设置PWM占空比，单位% |

### 5.3 数据帧率估算

9600 波特率下，每秒可发送 9600/10 = 960 字节。

一帧数据最长：`"255,255,255,255\n"` = 16字节。

理论帧率：960 / 16 ≈ **60 帧/秒**，满足波形显示需求。
